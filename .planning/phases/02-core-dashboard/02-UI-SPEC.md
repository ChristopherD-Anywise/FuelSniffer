# Phase 2 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — vanilla Tailwind CSS v4 |
| Preset | not applicable |
| Component library | none — hand-rolled Tailwind components |
| Icon library | none — text labels and CSS-only indicators only |
| Font | Geist Sans (already installed via `next/font/google`; `--font-geist-sans` variable in globals.css) |

Source: RESEARCH.md "Alternatives Considered" — explicit recommendation against shadcn for this project's zero-dependency preference.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, pill border radius padding |
| sm | 8px | Compact element spacing, gap between price and unit label |
| md | 16px | Default card padding, filter bar internal padding |
| lg | 24px | Station card padding, section gaps |
| xl | 32px | Layout column gap (list ↔ map) |
| 2xl | 48px | Page-level vertical padding |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets: minimum 44px height on all interactive elements (filter pills, sort toggle, slider thumb, station cards). Source: CONTEXT.md "usable in a parked car — large tap targets".
- Filter bar height: 56px fixed (accommodates pills + slider in one row on mobile).
- Map container: 100% of available viewport height minus filter bar (56px).

---

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display | 28px | 700 (bold) | 1.0 | Station price value (e.g. "145.9") — the dominant visual element in each card |
| Heading | 18px | 600 (semibold) | 1.2 | Station name |
| Body | 15px | 400 (regular) | 1.5 | Street address, suburb, distance in km |
| Label | 12px | 500 (medium) | 1.3 | Freshness indicator ("3 min ago"), fuel type pill text, unit label ("c/L") |

Notes:
- Display (28px bold) must pass WCAG AA at 4.5:1 against the card background in both normal and stale states.
- Stale card prices use Display size but at 40% opacity (Tailwind `opacity-40`) to communicate data age without removing the value.
- Font is Geist Sans for all roles. No monospace font in dashboard UI.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` (light) / `#0a0a0a` (dark) | Page background, full-bleed areas |
| Secondary (30%) | `#f4f4f5` (light) / `#18181b` (dark) | Station cards, filter bar background, map sidebar |
| Accent (10%) | `#2563eb` (blue-600) | Active fuel type pill background + text, active sort toggle, slider thumb, selected card highlight ring |
| Destructive | `#dc2626` (red-600) | Revoke invite code button only |

Accent reserved for:
1. Active/selected fuel type pill (background fill)
2. Active sort toggle button (background fill)
3. Distance slider thumb
4. Selected station card left border (4px left border when card is highlighted from map pin click)

Stale state: cards with price data older than 60 minutes receive `opacity-40` on the price display and address block. The card background remains secondary colour — no separate stale background token.

Map pin colour scale (Claude's Discretion — resolved here):
- Algorithm: HSL linear interpolation across the current result set min/max price.
- Cheapest station: `hsl(120, 70%, 35%)` (green)
- Median price: `hsl(55, 90%, 40%)` (amber)
- Most expensive: `hsl(0, 75%, 45%)` (red)
- Pin text: always `#ffffff` (white) at 700 weight for contrast against all pin colours.
- Pins use inline SVG `DivIcon` (Leaflet) — no PNG sprites required.

Source: CONTEXT.md "price pins should be immediately scannable"; RESEARCH.md "HSL lerp: green → yellow → red".

## Component Inventory

All components are hand-rolled Tailwind. No third-party component library.

### FilterBar (sticky, 56px height)
- Sticky top-0, z-10, full viewport width.
- Left section: fuel type pills — `FuelTypePills` component.
- Right section: distance slider — `DistanceSlider` component.
- Mobile (< md breakpoint): pills row above, slider row below = 88px stacked height.
- Background: secondary colour with 1px bottom border (`border-b border-zinc-200 dark:border-zinc-700`).

### FuelTypePill
- Height: 36px, horizontal padding: 16px, border-radius: 18px (pill shape).
- Inactive state: secondary background, body text colour, 1px border (`border-zinc-300`).
- Active state: accent fill (`bg-blue-600`), white text, no border.
- Touch target: 44px minimum via `min-h-[44px]` wrapper or `py-[4px]` adjustment.
- Pills in order: ULP91 | ULP95 | ULP98 | Diesel | E10 | E85.

### DistanceSlider
- Native `<input type="range">` styled via Tailwind.
- Range: 1–50km, step: 1km, default: 20km.
- Label: "Within [N] km" updates live as slider moves.
- Thumb: 24px circle, accent colour.
- Track: 4px height, accent colour for filled portion, secondary colour for remainder.

### StationCard
- Padding: 16px (md token).
- Layout: 3-column grid — [price block | station info | distance badge].
- Price block: Display 28px bold price + Label 12px "c/L" unit right-aligned to price.
- Station info: Heading 18px station name + Body 15px address + Label 12px freshness.
- Distance badge: Body 15px "[N.N] km" right-aligned.
- Stale state: price block + address block at `opacity-40`; freshness label shows stale indicator text.
- Selected state (from map pin click): 4px left border in accent colour.
- Card height: minimum 80px (auto-grows for long station names).
- Touch target: entire card is tappable (triggers map pin highlight).

### StationList
- Sorted list of `StationCard` components in a scrollable container.
- Desktop: fills left half of split view, independent scroll.
- Mobile: full viewport width, scrollable.

### MapView
- Loaded via `dynamic(() => import('./MapView'), { ssr: false })` — mandatory for Leaflet.
- Default centre: North Lakes (`-27.2353, 153.0189`), zoom 12.
- Tiles: OpenStreetMap standard (`https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png`).
- Pin: custom Leaflet `DivIcon` — coloured circle (32px diameter) with white price text at 12px 700 weight.
- Desktop: fills right half of split view.
- Mobile: hidden by default; toggle button in filter bar switches between list and map views.

### SplitView layout
- Desktop (md+): `grid grid-cols-2`, full viewport height minus filter bar.
- Mobile (< md): `StationList` shown by default. Toggle button in `FilterBar` switches to `MapView`.
- Toggle button label: "Map" (list active) / "List" (map active). Position: right end of filter bar.

### LoadingSkeleton (loading state)
- 5 skeleton cards rendered during initial data fetch.
- Skeleton mimics `StationCard` dimensions: same 80px height, grey placeholder blocks for price and text.
- Uses Tailwind `animate-pulse` on `bg-zinc-200 dark:bg-zinc-700` blocks.
- No spinner — skeleton only.

### EmptyState
- Shown when API returns zero stations for the current fuel + radius combination.
- See Copywriting Contract for copy.
- Layout: centred in the list column, vertically centred.
- No illustration — text only.

### ErrorState
- Shown when the `/api/prices` fetch fails (network error or non-200 response).
- See Copywriting Contract for copy.
- Includes a "Try again" button (accent colour, 44px height) that retries the request.

### LoginPage
- Single input: invite code (text, `<input type="text" autocomplete="one-time-code">`).
- CTA button below input.
- Centred card layout, 320px max width.
- No logo — app name "FuelSniffer" as Heading text above input.

## Interaction Contract

### Filter interactions
- Selecting a fuel pill: immediate re-fetch of `/api/prices?fuel={id}&radius={current}`. URL param `fuel` updates. Loading skeleton shown during fetch.
- Moving distance slider: debounced 400ms before re-fetch. URL param `radius` updates. Label updates live with every slider tick.
- Both params reflected in URL — browser back/forward works. Shareable link contains current filter state.

### Sort toggle
- Two states: price-asc (default) and distance-asc. Toggle button switches between them.
- Sort applied client-side to the fetched result set — no new API call.
- URL param `sort=price|distance`.

### Card ↔ map sync (D-08)
- Clicking a station card: corresponding map pin animates to "selected" state (white ring around pin, pin scales to 1.2x). Map pans to centre on that pin.
- Clicking a map pin: corresponding card scrolls into view in the list, card receives selected 4px left border.
- At most one station selected at a time. Clicking empty map area deselects.

### Mobile map toggle
- Toggle button in filter bar. One panel visible at a time.
- List is default on page load.
- Selected station state persists across panel toggles (card highlighted when switching back to list).

### Login flow
- User enters invite code, presses "Enter dashboard" or hits Enter key.
- On success: redirect to `/dashboard` with filter defaults (ULP91, 20km).
- On failure: inline error below input — "That code isn't valid. Check with the person who shared it."
- Session cookie: HttpOnly, 7-day expiry, Secure flag (requires HTTPS via Caddy).
